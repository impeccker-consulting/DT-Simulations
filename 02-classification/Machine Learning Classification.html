What is Classification?

Classification assigns a label (a category) to a new input based on patterns learned from labelled examples. It is the most commonly deployed form of machine learning in business.

Foundations · ~10 mins

The Core Idea: Sorting Into Buckets

Imagine you are a doctor examining patient files. For each one you say YES (will develop diabetes) or NO (will not). That is classification. Your years of experience with past patients is the training data. The patient file is the new input. Your diagnosis is the prediction.

In ML we formalise this: given features X (age, blood sugar, BMI), a model learns a decision boundary separating YES from NO. For a new patient we measure where they fall relative to that boundary and output a probability — then compare it to a threshold to make the final call.

Binary ClassificationTwo possible outputs: Spam or not spam. Fraud or not fraud. Churn or not churn. About 90% of real-world ML deployments are binary classification problems.
Multi-class ClassificationThree or more labels. Which product category? Positive / Neutral / Negative sentiment? Which disease? Each input gets exactly one label from a fixed set.
What the Model LearnsA boundary (line, curve, or complex surface) in feature space that separates classes. Different algorithms draw this boundary in different ways — that is what this entire session explores.
Output is a ProbabilityMost classifiers say "73% chance of YES". A threshold (often 50%) converts that probability into a hard label. The threshold is a business decision, not a mathematical one — it determines the trade-off between catching more positives vs avoiding false alarms.

📌 Instructor note: Ask the class to name 5 classification problems they encounter before breakfast: unlock screen (face recognition), spam filter, news feed curation, Paytm fraud alert, Ola driver safety score. Classification is everywhere.

🧠 The Human Analogy

A loan officer sees an application: income ₹8L/year, age 32, existing loans ₹2L, owns a house. She says APPROVED. Her mental model was built from thousands of past decisions — she has learned where the boundary between safe and risky borrowers lies.

🤖 The ML Equivalent

A classification model reads the same fields. Trained on 50,000 historical decisions, it outputs P(default) = 0.12 → APPROVED. Faster, consistent, auditable, and processes 10,000 applications per second without fatigue or bias from the last case reviewed.

🏦

Real Story: HDFC Bank Credit Scoring

HDFC Bank processes over 4 lakh credit card applications per month. Human underwriters used to take 3–5 days per application. Their XGBoost-based classification model — trained on income, existing credit, transaction history, and employment stability — now classifies 90% of applications instantly with no human review.

The model outputs P(default in 12 months). If this exceeds a threshold set by the credit policy team, the application is flagged for review. Lowering the threshold = conservative book, fewer approvals. Raising it = growth mode, higher risk. The threshold is a business decision — not something the algorithm decides. This is why understanding classification is inseparable from understanding business trade-offs.

Logistic Regression

Despite the name, Logistic Regression is a classification algorithm. It estimates the probability that an input belongs to a class using the sigmoid (S-shaped) function.

Algorithm 1 of 7 · ~15 mins

The S-Curve That Changed Everything

Linear Regression predicts numbers from −∞ to +∞. For classification we need a probability between 0 and 1. Logistic Regression squashes any number into (0,1) using the sigmoid function — think of it as bending a straight line into an S-curve.

Any input far to the left of the S gets probability near 0 (legit). Any input far to the right gets probability near 1 (fraud). The middle zone is genuine uncertainty — this is where the threshold matters most.

StrengthsExtremely fast to train. Probabilities are directly interpretable. Coefficients reveal feature importance and direction. Works well when classes are roughly linearly separable. Always the correct first baseline to build.
WeaknessesCannot learn non-linear boundaries on its own. Assumes a linear relationship between features and log-odds. Struggles when complex feature interactions exist (e.g. "elderly AND high income" is very different from either feature alone).
Indian Use CasesCIBIL score to loan approval. Zepto: will this customer churn this week? Tata AIG: will this vehicle insurance claim be fraudulent? Biocon: will this drug compound bind to the target protein?
When to Use ItFirst attempt on any new classification problem. When predictions must be explainable to regulators (RBI, SEBI, IRDAI). When training data is small. When real-time scoring at microsecond speed is required.

🔬 The Sigmoid Formula — What Does σ(z) = 1 / (1 + e^−z) Mean?

P(Y=1 | X) = σ(z) = 1 / (1 + e^−z) where z = w₀ + w₁·X₁ + w₂·X₂ + ... + wₙ·Xₙ
P(Y=1|X)
Probability the output is Class 1 given input X. For HDFC: P(default | income, age, debt). This is what the model returns — a number between 0 and 1, before any threshold is applied.
z = w₀+wᵢXᵢ
A plain weighted sum of features — exactly like Linear Regression. Positive weight = pushes towards Class 1 (fraud). Negative weight = pushes towards Class 0 (legit). These weights are learned from training data.
e^−z
e ≈ 2.718 (Euler's number). When z is large positive, e^−z ≈ 0 → probability ≈ 1. When z is large negative, e^−z is huge → probability ≈ 0. The maths automatically keeps output inside [0,1] without any additional constraints.
Threshold τ
The business decision layer. If P(fraud) > τ, flag the transaction. Default τ = 0.50. A bank might use τ = 0.30 for higher recall (catch more fraud). An e-commerce site might use τ = 0.70 to avoid annoying legitimate customers.
💡 Plain English: Multiply each feature by a learned weight and add them up. Very positive sum → high fraud probability. Very negative sum → low probability. The sigmoid squashes the sum to 0–1. Compare to threshold to decide.

Decision Trees & Random Forest

Decision Trees ask a series of yes/no questions. Random Forests ask hundreds of trees and take a majority vote. One is transparent; the other is powerful.

Algorithms 2 and 3 of 7 · ~20 mins

Decision Trees: How a Machine Plays 20 Questions

A Decision Tree splits data by asking the best question at each step — "Is amount > ₹10,000?" splits into two buckets. It picks the question that creates the purest buckets (one mostly fraud, the other mostly legit). It keeps splitting until purity stops improving or a depth limit is reached.

The purity measure is called Gini Impurity (or Information Gain / Entropy). A perfectly pure bucket has Gini = 0. A 50/50 split has Gini = 0.5 — maximum impurity. The algorithm tries every possible split on every feature and picks the one that reduces average impurity the most.

Decision Tree ProsHumans can read every rule. No feature scaling needed. Handles numeric and categorical features. Fast inference. Ideal where decisions must be auditable (RBI, IRDAI, HR tribunals).
Decision Tree ConsHighly prone to overfitting — trees memorise noise when grown deep. Unstable: small data changes can produce entirely different trees. Single trees are weak learners on complex problems.
Random Forest ProsBuilds hundreds of trees on random data subsets and feature subsets. Ensemble dramatically reduces overfitting. Robust to outliers and missing values. Built-in feature importance. Industry workhorse for structured data.
Random Forest ConsBlack box at individual prediction level. Slower and more memory-intensive than a single tree. Storing hundreds of trees requires significant RAM. Struggles with very sparse, high-dimensional text data.

🔬 Gini Impurity — The Question-Picker Formula

Gini(node) = 1 − Σ pᵢ² where pᵢ = fraction of samples in class i at this node
Pure node
100 transactions, all fraud. p_fraud = 1.0. Gini = 1 − (1.0²) = 0. Perfect purity — tree stops here.
50/50 split
50 fraud, 50 legit. Gini = 1 − (0.5² + 0.5²) = 0.5. Maximum impurity — must split further.
After split
"Amount > ₹10,000?" Left: 80 fraud/20 legit (Gini=0.32). Right: 10 fraud/90 legit (Gini=0.18). Weighted avg = 0.26. Gain = 0.50 − 0.26 = 0.24 — great split, tree picks it.
Plain English
Try every possible question on every feature. Pick the one that most cleanly separates fraud from legit. Gini measures how clean the separation is. Repeat at each node until leaves are pure or depth limit is hit.

Brain Teaser: 100% Training Accuracy — Good or Bad?
An unconstrained Decision Tree achieves perfect training accuracy. Should you deploy it?
+20 pts▼

An Ola data scientist builds a Decision Tree (no depth limit) to predict driver safety. Training accuracy = 100%. Test accuracy = 62%. A Random Forest with 200 trees scores Training = 97%, Test = 89%.

What happened to the tree, and why does RF fix it?

A
The tree learned perfectly — 100% training accuracy means it is correct. The test set must be unrepresentative.
B
The tree overfitted — with no depth limit it memorised every training point including noise. RF fixes this by training each tree on a different random 63% bootstrap sample and random feature subset, so no tree can memorise everything. Aggregating diverse trees cancels out individual errors.
C
The tree needs more data — 100% training accuracy with low test accuracy always means insufficient data.
D
Both models are wrong — always use Logistic Regression before trying tree-based models.

SVM, K-Nearest Neighbours & Naive Bayes

Three fundamentally different philosophies: maximum margin separation, neighbourhood voting, and probabilistic reasoning. Each excels in specific domains.

Algorithms 4, 5 and 6 of 7 · ~20 mins

Three Different World Views on Classification

SVM asks: "What is the widest possible road I can draw between the two classes?" It finds the hyperplane (decision boundary) that maximises the margin — the gap between the closest points of each class. Only the points on the edge of the margin matter — the support vectors. Remove everything else and the boundary is unchanged.

KNN asks: "Who are this point's closest neighbours and what class are most of them?" No training phase — the model stores all training data. At prediction time it finds the K nearest points by distance and takes a majority vote. Simple, intuitive, no assumptions.

Naive Bayes asks: "Given these features, which class is statistically most likely?" It applies Bayes' Theorem with the naive assumption that all features are independent of each other. Wrong in most real datasets, but it works surprisingly well for text because relative probabilities still align.

SVM: Best ForHigh-dimensional data (text, images). Clear margin between classes. Small-to-medium datasets. Medical imaging (tumour vs normal), email spam, face recognition. The kernel trick handles non-linear boundaries.
KNN: Best ForSmall datasets. Recommendation systems ("customers like you also bought..."). Anomaly detection. Exploratory analysis when you need a fast non-parametric baseline before committing to a heavier model.
Naive Bayes: Best ForText classification (spam, sentiment, topic labelling). Real-time classification at internet scale. Multi-class problems. When training data is limited — NB generalises well from few examples, making it ideal for new problem domains.
Key Trade-offsSVM: slow to train at large scale but excellent on text. KNN: no training cost but prediction slows linearly with data size. NB: fastest of all but independence assumption is often wrong. Always benchmark all three before committing to production.

🔬 SVM: Maximum Margin — The Widest Road Formula

Maximise: 2 / ||w|| Subject to: yᵢ(w·xᵢ + b) ≥ 1 for every training point i
w·x + b = 0
The decision boundary (hyperplane). In 2D this is a line. In 100 dimensions (text features) it is a hyperplane. Points on one side are Class 0, the other side are Class 1.
Margin = 2/||w||
The width of the road between classes. SVM maximises this. A wider margin means more confident predictions on unseen data — the boundary is as far as possible from both classes.
Support Vectors
The training points on the edge of the margin. These are the hardest cases to classify. They alone define the boundary. All other points are irrelevant to the model — removing them changes nothing.
Kernel Trick
When classes are not linearly separable, the kernel maps data to higher dimensions where a linear boundary IS possible. RBF kernel maps 2D data to infinite dimensions — like uncrumpling a sheet to separate two groups of dots.

🔬 Naive Bayes: Bayes' Theorem Applied to Text

P(Spam | words) ∝ P(Spam) × P("free"|Spam) × P("win"|Spam) × P("prize"|Spam)
P(Spam|words)
What we want to know. Given that this email contains "free", "win", "prize" — how likely is it spam? The model computes this from word frequency counts learned from millions of training emails.
P(word|Spam)
Likelihood: how often this word appears in spam. In training data, "free" appeared in 80% of spam but only 5% of legitimate emails. This ratio makes "free" a very strong spam signal. NB learns thousands of such ratios.
P(Spam)
Prior probability. What fraction of all training emails were spam? Say 30%. This is the baseline before looking at any words. Bayes' theorem updates this prior with evidence from each word in sequence.
Naive Part
We assume "free", "win", "prize" are independent of each other. Mathematically false — spammers use them together. But the model works because relative class probabilities align correctly even when the independence assumption is violated.

XGBoost — The Kaggle King

XGBoost builds trees sequentially, each correcting the errors of all previous trees. It is the single most effective algorithm for structured tabular data and powers fraud detection across India's top fintech companies.

Algorithm 7 of 7 · ~15 mins

Boosting: Learning From Mistakes, One Tree at a Time

Random Forest builds hundreds of trees in parallel and averages them. XGBoost builds trees in sequence. Tree 1 makes predictions. Tree 2 focuses entirely on the cases Tree 1 got wrong. Tree 3 focuses on what Tree 2 still gets wrong. After 100–1000 trees, the combined model is extraordinarily accurate on structured data.

Each new tree is fitted to the residual errors — the gap between current predictions and true labels. This is gradient descent applied to function space: the algorithm uses calculus to find the direction that reduces error fastest, then builds a tree to step in that direction.

Why XGBoost WinsL1 and L2 regularisation built in. Handles missing values natively. Custom loss functions for imbalanced data (scale_pos_weight). Parallelised tree building despite sequential logic. SHAP explainability built in.
Real Deployments in IndiaFlipkart: fraud scoring on 10M+ daily transactions. Zerodha: options manipulation detection. Apollo Hospitals: readmission risk. PhonePe: AML flagging. 17 of 29 Kaggle winners (2015–2017) used XGBoost.
Key Hyperparametersn_estimators: number of trees (100–1000). max_depth: depth per tree (3–8). learning_rate/eta: step size per tree (0.01–0.3). subsample: fraction of data per tree (0.6–1.0). Lower eta + more trees = better but slower.
When NOT to Use XGBoostWhen full transparency is required (use LR or Decision Tree). When data is under 500 rows (overfits easily). When inputs are images or text sequences (use CNNs or Transformers). For everything else tabular: try XGBoost.

🔬 Gradient Boosting Formula — Building on Mistakes

F_m(x) = F_{m-1}(x) + η × h_m(x) where h_m is trained on residuals rᵢ = yᵢ − F_{m-1}(xᵢ)
F_{m-1}(x)
The current ensemble prediction. At step 0 this is just the base rate (e.g. 30% fraud). At step 100 it is the combined output of 100 correction trees stacked together.
rᵢ = yᵢ−F_{m-1}
The residual error for each training point. "My model predicts P(fraud)=0.30 but this was actually fraud (1.0). Residual = 0.70." Tree m is trained to predict these residuals — focusing effort on the hardest cases.
h_m(x)
The new tree fitted to residuals. This tree learns where the current ensemble is wrong and by how much. Training effort concentrates on cases where the gap between prediction and ground truth is largest.
η (eta)
Learning rate — how cautiously each tree contributes. η = 0.1 means each tree adds only 10% of its correction. Lower eta = more trees needed, but more stable and usually better final accuracy. Prevents overcorrection.

Which Classifier Do I Use?

There is no universally best algorithm. The right choice depends on data type, size, interpretability requirement, latency constraint, and the cost of different error types.

Synthesis · ~10 mins

The Algorithm Selection Framework

A practical heuristic: Start with Logistic Regression. If it cannot capture the pattern, try Random Forest. If you need the last 2–3% accuracy on tabular data, try XGBoost. For real-time text classification, use Naive Bayes. For high-dimensional feature spaces with a clear margin, try SVM.

What matters most is not the algorithm — it is asking the right questions first: Who must explain this decision? How fast must it run? How much data exists? What type of data is it? The answers constrain your choice more than accuracy benchmarks do.

Rule 1: InterpretabilityRegulatory requirement (RBI, SEBI, IRDAI, HR tribunal)? Use Logistic Regression or Decision Tree. "Black box is fine, optimise accuracy"? Use XGBoost or Random Forest.
Rule 2: Data SizeUnder 1,000 rows → LR or SVM. 1,000–100,000 rows → RF or XGBoost. Over 1M rows with text → NB or fine-tuned deep learning. KNN does not scale past ~50,000 rows without approximation tricks.
Rule 3: Data TypeTabular business data → LR then RF then XGBoost ladder. Text and NLP → NB then SVM then BERT. Images → CNNs exclusively. Mixed tabular and text → separate pipelines, then combine predictions.
Rule 4: Class ImbalanceSevere imbalance (fraud = 0.1% of data)? All classifiers need SMOTE, class weights, or threshold adjustment. XGBoost handles this best via scale_pos_weight. Logistic Regression is most sensitive to raw imbalance without correction.

Stacking Challenge — When Do Ensembles of Classifiers Win?
Razorpay combines LR + RF + XGBoost and beats any single model. Why?
+20 pts▼

Razorpay trains: LR (AUC=0.82), RF (AUC=0.88), XGBoost (AUC=0.91). They build a stacking ensemble — a meta-model that takes all three outputs as inputs. The stacked model scores AUC=0.94. Why does stacking work, and when does it NOT?

A
Stacking always works — combining more models always improves accuracy because errors mathematically cancel out.
B
Stacking works when base models make DIFFERENT errors (diverse failures). LR misses non-linear patterns. RF misses sequential time-based fraud. XGBoost covers most structured patterns. The meta-model learns when to trust which classifier. Stacking FAILS when all base models fail on the same cases — correlated failures add no new information.
C
Stacking works because averaging probabilities always reduces variance regardless of model correlation.
D
Stacking only works with exactly three classifiers — a fourth model always degrades performance.
